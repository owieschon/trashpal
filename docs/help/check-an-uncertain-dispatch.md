# Check an uncertain dispatch

Use this page when a dispatch operation is marked unknown because TrashPal cannot tell whether the outside dispatch system accepted it.

## Treat unknown as unresolved

An unknown status is not a failed dispatch, and it is not a completed collection. It means the original request may have been accepted, but TrashPal did not receive enough information to know the result.

Do not submit the recovery again. A second submission could create duplicate work.

## Check the existing operation

Open the operation already attached to the case and use its status check. TrashPal looks for the matching assignment using the original recovery, rather than preparing another recovery.

The check can lead to one of these outcomes:

- The assignment is found, so the operation moves forward with the existing work.
- The assignment is not found, so the case remains unresolved and needs the appropriate operating follow-up.
- The outside system still cannot be checked, so the operation remains unknown until there is better information.

Keep the original case and operation together while it is being checked. That preserves the record of what was approved and prevents a duplicate recovery from looking like a new request.

When the operation has new outcome information, continue to [read a recovery receipt](read-a-recovery-receipt.md).
